# Release Checklist

Use this checklist for Windows beta releases published through GitHub Releases.

## Versioning

Update the app version in:

- `package.json`
- `src-tauri/Cargo.toml`
- `src-tauri/tauri.conf.json`

Use the same version number in all three places.

## Pre-Release Validation

Run:

```bash
npm install
npm run build
cargo check --manifest-path src-tauri/Cargo.toml
npm run tauri:build
```

Smoke-check the packaged app:

- installer launches
- app opens successfully after install
- street and satellite maps load
- region packs appear after being copied into the offline maps directory
- live flight start/end flow works
- saved-flight review works

## Release Assets

Primary asset:

- Windows NSIS installer generated by `npm run tauri:build`

Optional separate assets:

- region-pack `.zip` files
- release notes or mission-packaging notes if needed

## GitHub Release Flow

1. Commit the version bump.
2. Create and push a tag in the form `vX.Y.Z`.
3. Let the GitHub Actions workflow build the Windows installer.
4. Verify that the release contains the installer asset.
5. Upload any region-pack archives separately if they are part of the release.

The workflow can also be run manually with `workflow_dispatch` to produce a build artifact without publishing a tagged release.

## Release Notes Template

Use this structure:

```md
## Download

- Windows installer: use the `.exe` asset in this release
- Region packs: download separately if provided

## What's new

- Bullet summary of operator-visible changes

## Known issues

- Bullet summary of active beta limitations

## Install notes

- This beta installer is currently unsigned, so Windows SmartScreen may warn before launch
- Offline map regions are installed separately under `%LOCALAPPDATA%\\com.kerbodyne.groundstation\\offline-maps\\`
```
